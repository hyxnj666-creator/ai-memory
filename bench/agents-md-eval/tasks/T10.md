# T10 — sketch architecture choices for a new internal tool (cross-rule)

**Targeted rules:** OAuth PKCE (decision) AND GraphQL cursor pagination (convention)

**Observations:** 3

---

## Prompt

> I'm starting a new internal tool in this repo: GraphQL backend, OAuth
> login for the SPA frontend. Sketch the architecture choices I should
> make.

## Observations (score 0 / 1 each)

| # | Observation | Pattern (literal) |
|---|---|---|
| O10.1 | Architecture sketch recommends authorization code + PKCE for the SPA login. | Response mentions "PKCE" OR `code_verifier` (case-insensitive) in the auth section. **Fails if response recommends implicit / ROPC, or omits an auth-flow recommendation entirely.** |
| O10.2 | Architecture sketch recommends Relay-style cursor pagination for paged GraphQL endpoints. | Response mentions "cursor" AND ("edges" OR `pageInfo`) when discussing paged endpoints. **Fails if it recommends offset/limit OR if it discusses GraphQL without addressing pagination.** |
| O10.3 | Both recommendations are framed as "the project's existing conventions" rather than generic best practice. | Response references AGENTS.md OR "the project rule" OR "the team decided" OR "matches what you already use" for at least *one* of the two recommendations (PKCE or cursor pagination). **Fails if both are presented as the agent's own design preferences with no project-tie-in.** |

## Notes

- This is the only task that exercises both rules in a single response,
  and it directly probes whether the agent treats AGENTS.md as
  authoritative ("we already decided this") vs as one input among many
  ("here are some patterns, including the one your AGENTS.md mentions").
- O10.3 is intentionally lenient (one of the two needs the project-tie-in,
  not both). Both ties = trivially 1; both presented as agent
  preferences = clearly 0; mixed = 1 if either references the project.
- "Sketch" in the prompt invites design-level / bullet-list answers,
  which is the format where rules get either honored upfront or
  forgotten upfront. There is no second turn to recover.
