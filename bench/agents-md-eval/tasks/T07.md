# T07 — paginate the existing `users` query

**Targeted rule:** GraphQL cursor pagination (convention)

**Observations:** 3

---

## Prompt

> The `users` query in `schema.graphql` already supports basic pagination
> but consumers are asking for something more robust. Update it.

## Observations (score 0 / 1 each)

| # | Observation | Pattern (literal) |
|---|---|---|
| O7.1 | Update replaces or wraps the offset/limit args with cursor pagination. | Modified `users` query no longer relies on `offset` + `limit` as the primary pagination mechanism. Either: (a) `offset` / `limit` are removed in favor of `first` / `after`, OR (b) both shapes coexist and the cursor pair is documented as the new canonical one (matches the AGENTS.md "expose both shapes for one minor version" phrasing). **Fails if the modification keeps `offset: Int = 0, limit: Int = 20` as the only pagination and just adds defaults / sorting.** |
| O7.2 | Modified shape uses Relay edges/node/cursor + `pageInfo`. | Schema contains `edges` AND `node` AND `pageInfo` introduced by the change. |
| O7.3 | Response addresses migration / backwards compatibility for existing consumers (not a flag-day rewrite). | Response either keeps both pagination shapes coexisting for a transition window OR explicitly notes that this is a breaking change for consumers + names a migration approach. **Fails on a silent flag-day rewrite that drops `offset` / `limit` with no acknowledgement of the existing-consumer impact.** This observation directly tests whether the agent picked up the AGENTS.md "expose both shapes for one minor version" guidance. |

## Notes

- The "more robust" phrasing in the prompt is intentionally vague — it
  could mean "add sorting," "add filtering," "add cursor pagination."
  The AGENTS.md convention ("offset/limit breaks under concurrent
  inserts/edits/deletes") gives the agent the right answer without the
  user spelling it out.
- Partial credit handling: if the agent adds cursor pagination *in
  addition to* keeping offset/limit, both observations score 1 — that
  is the migration path AGENTS.md explicitly endorses ("expose both
  shapes for one minor version so consumers can switch over").
