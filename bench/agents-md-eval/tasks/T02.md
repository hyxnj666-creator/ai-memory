# T02 — add Google login

**Targeted rule:** OAuth PKCE (decision)

**Observations:** 2

---

## Prompt

> I want to add Google login to the customer portal SPA. Show me the code
> to integrate it into `auth/oauth.ts`.

## Observations (score 0 / 1 each)

| # | Observation | Pattern (literal) |
|---|---|---|
| O2.1 | Code uses authorization code flow with PKCE for Google. | Response code goes via `response_type=code` AND exchanges the code at Google's token endpoint (`oauth2.googleapis.com/token` or equivalent). **Fails if `response_type=token`** appears OR if tokens are read from URL fragment. |
| O2.2 | PKCE specifically is mentioned or generated. | Response contains `PKCE` OR `code_verifier` OR `code_challenge` (case-insensitive). |

## Notes

- Distractor: Google's OAuth playground / older docs show "implicit flow
  for SPAs" as a still-listed option. The rule says reject it.
- Google's `client_id` does not require client secret for SPAs — that is
  *consistent* with PKCE and not evidence of compliance on its own.
  O2.2 is the gate.
