### PR title

```
Add ai-memory-cli to known generators (chat-history → AGENTS.md)
```

### PR body

```markdown
Adds [`ai-memory-cli`](https://github.com/hyxnj666-creator/ai-memory) (npm: [ai-memory-cli](https://www.npmjs.com/package/ai-memory-cli)) to the known-generators registry.

## What it does

Reads chat history from Cursor, Claude Code, Windsurf, VS Code Copilot Chat, and OpenAI Codex CLI — five editors that all natively read or are part of the AGENTS.md ecosystem — classifies the content into five typed memory categories (decision / architecture / convention / TODO / issue) using an LLM extraction pass, and emits an AGENTS.md section via an idempotent managed-section marker:

```markdown
<!-- ai-memory:managed-section start -->
... auto-generated, only this region is rewritten on re-runs ...
<!-- ai-memory:managed-section end -->
```

Hand-written content outside the marker is preserved byte-for-byte across re-runs. The same writer also handles "no markers found → append at bottom", "markers already present → in-place section replacement", and "malformed markers → refuse with a clear error" (cf. the test suite at [`src/__tests__/agents-md-writer.test.ts`](https://github.com/hyxnj666-creator/ai-memory/blob/main/src/__tests__/agents-md-writer.test.ts)).

## Why it might fit "known generators"

- It's the only published tool we're aware of whose primary input is editor chat history rather than `add()` / `remember()` API calls (cf. [competitive-landscape.md](https://github.com/hyxnj666-creator/ai-memory/blob/main/docs/competitive-landscape.md), Bucket 1).
- It exposes a published benchmark (CCEB — Cursor Conversation Extraction Benchmark) measuring extraction quality on `gpt-4o-mini`: F1 76.2% / Precision 66.7% / Recall 88.9% over 9 hand-curated fixtures (v2.5-01 baseline, 2026-04-26).
- The AGENTS.md writer respects the spec's hand-edit-friendly intent — the managed-section marker keeps the file PR-reviewable and merge-conflict-friendly.

## Suggested entry

```markdown
- [ai-memory-cli](https://github.com/hyxnj666-creator/ai-memory) — Generates AGENTS.md from editor chat history (Cursor / Claude Code / Windsurf / Copilot / Codex CLI). Idempotent managed-section markers; plain Markdown memory source under git. MIT licensed.
```

Happy to adjust wording / placement / category to match the existing entries — please point me at the right section if this PR landed in the wrong place.
```

### 6.5 Canonical "what's the differentiator" claim (from competitive-landscape.md)

When asked / when the listing has space for it, the four claims are:

1. Zero `.remember()` boilerplate — input is editor chat history we already have.
2. Native AGENTS.md output with idempotent managed-section markers.
3. Plain Markdown in git, no database — `.ai-memory/` is the source of truth.
4. Time-travel recall via git history — `ai-memory recall <query>` shows every prior version of every memory.

Listings that allow more than 1 paragraph should pick claims 1 + 2 (the
distribution surfaces — every reader gets these). Listings about
"developer experience" specifically should add claim 4.
